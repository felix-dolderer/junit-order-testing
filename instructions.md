# Instructions

## Goal
- Setup a Kotlin project with JUnit5 as the testing framework
- Showcase a test, with a specified order of execution (using OrderAnnotation configured as a global default) 

## Background / Motivation
- In our main project, we currently use testNG, but want to move to JUnit5.
- Before we move, we need to demonstrate a suitable replacement for the dependsOnMethod functionality in testNG.
- This repo is supposed to be that showcase.

## Additional Information
- The actual content / functionality in the repo / under test is not relevant. Just make sure it's comprehensible.
- Add necessary dependencies, but only the necessary dependencies
